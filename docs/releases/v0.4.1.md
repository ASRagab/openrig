# OpenRig v0.4.1

## Summary

OpenRig v0.4.1 is an observability + operator-UI overhaul: the workspace
becomes a developer's mission cockpit. Across the Project UI, altitude
projection (workspace → mission → slice) and a coherent tab set replace
the partially-built Project pages from v0.4.0. The Mission landing tab
shows steering + the human-facing mission brief. The Story tab
reconstructs what actually happened on a mission as a scrollable
git-graph derived from queue lineage. The Workflow tab visualizes the
configured workflow spec. The Artifacts navigator + Proof tab + Progress
heat-map give workspace altitude its own native surfaces. The Dashboard
route gets a clean cosmetic refresh as the launcher home.

Underneath: a UI digital twin renders all six real Project surfaces
1:1 daemon-free from typed fixtures so visual intent can be approved
before any UI slice is built, then proven against a real screenshot at
the end. A topology edge-flow animation surfaces real queue/handoff
movement on the topology graph (signal-driven, never decorative). The
For-You feed picks up a phone-friendly restyle with real-data fidelity.

New CLI: `rig auth` lands as a product-native, CLI-local Codex
auth-profile manager with a hardened secret boundary that refuses
symlink + hardlink escapes out of CODEX_HOME. `rig send` ships a 3-layer
interactive-prompt guard (default-blocked at a target pane's prompt;
`--raw` keystrokes still guarded; `--dangerously-interact --reason`
audited override) so a peer message can no longer accidentally select
an option in another agent's terminal.

Reliability fixes: a fresh `npm install -g @openrig/cli` followed by
`rig daemon start` now resolves cleanly (a missing `@hono/node-ws`
runtime dep was masked by monorepo hoisting on the dev host but failed
on a fresh install; the CLI now mirrors the daemon's full runtime
dep set, gated by a packaging-completeness test). Table-view + hybrid
terminal-action errors surface as visible alerts instead of silent
failures, and the table terminal opens. Queue items take a
human-readable `summary` field (additive migration 044; phased
WARN-on-author).

Existing v0.4.0 databases upgrade by running `rig daemon start` on the
new daemon. Migrations are additive (044 adds a nullable
`queue_items.summary` column; pre-0.4.1 items degrade cleanly to a
body-fallback label).

## Included In This Release

### New top-level CLI surface

- **`rig auth`** — product-native, CLI-local Codex auth-profile
  management. List / show / switch / capture / forget Codex profiles
  under a CLI-owned `~/.openrig/codex-auth/`, never reaching outside
  CODEX_HOME. The adversarial path is closed: a symlinked active auth
  is refused, hardlinks and symlinked parents cannot escape CODEX_HOME,
  and the secret boundary is integration-tested. Replaces the prior
  ad-hoc out-of-product Codex auth swap with a single first-class
  surface.

### `rig send` — interactive-prompt guard

A peer message can no longer accidentally pick an option or approve a
permission in another agent's terminal. Three layers:

- **L0 (default)** — `rig send` is **blocked** when the target pane is
  at an interactive prompt or permission block. Detection is
  hook-primary (Codex `PermissionRequest`) with a hardened capture-pane
  fallback (12-line trailing-content scan + 15-second send-readiness
  freshness).
- **L1 — `--raw`** — exact keystrokes; the L0 guard still applies.
- **L2 — `--dangerously-interact --reason "..."`** — explicit override,
  implies `--raw`, requires `--reason`, writes an audit row.

This closes the long-standing "stand-down message submitted an
authorize-publish default" footgun.

### Workspace observability tabs (the headline)

- **Steering tab (mission landing)** — Panel 1 projects `STEERING.md`
  (what agents are currently being told to do); Panel 2 projects the
  mission's human-facing `MISSION_BRIEF.md` derived from the new locked
  7-section schema. The landing tab when you click into a mission;
  Overview stays available.
- **Mission brief doc-type + projection** — a real canonical
  human-facing doc-type alongside `README.md` / `PROGRESS.md` /
  `MISSION_NOTES.md`. `rig scope init-workspace` /
  `rig scope mission create` emit a root `MISSION_BRIEF.md` from the
  7-section template (Brief / What & why / Building / Progress / Proven
  / Needs you / Pointers); backfill the same template into existing
  missions on demand. The Mission Steering tab renders it
  markdown → UI.
- **Story tab — queue-item DAG git-graph** — reconstructs a forest of
  DAGs from `chain_of_record` + handoff lineage as a scrollable,
  upward-growing git-graph. Single-parent edges only (no invented merge
  state); one-line rows over a curved gutter; full-width expanded row;
  a Tier-3 drawer reuses the queue-item viewer for the full agent-speak
  body + chain. Both mission and slice altitudes; client-side from the
  existing `/api/queue` read API (no new surface). The Queue tab folds
  into Story.
- **Workflow tab — spec visualizer** — finishes the half-built Project
  Topology tab into a real read-only visualizer of the configured
  workflow spec: dotted-grid canvas, dark-header step cards, per-step
  state dot, dagre LR layout, amber reject → rework loop-back. Label
  rename Topology → Workflow (test-id stays `project-tab-topology`,
  id-stable). Both mission and slice altitudes. Workspace-only; the
  standalone Topology page (NOC) is unchanged.
- **Proof tab** — per-slice proof galleries + empty-state for
  scaffolded-but-unpopulated slices. Reads from the slice's
  `proof/` directory + a `PROOF.md` summary if present.
- **Progress heat-map** — Project Progress tab consolidates to the
  heat-map (per-slice rollup cards retired); the heat-map carries
  status at a glance without the duplicate-card noise.
- **Artifacts navigator** — slice-altitude `ArtifactsNavigator`
  replaces the prior `SliceArtifactsTab` card wall; files and docs
  resolve through the same `/api/files/*` surface; the Proof tab owns
  proof artifacts; the Decisions surface routes through Story
  (decision-of-record items are already Story nodes).
- **Workspace portfolio** — workspace parent altitude renders a
  cross-mission portfolio panel: mission list with steering glance on
  expand, lazy `MISSION_BRIEF.md` fetch, rollup metric counts
  *proven* (from `hasProofPacket`) rather than `done`.

### UI digital twin + visual intent → proof convention

- **UI digital twin (harness)** — a derive-from-source twin renders all
  six real Project UI surfaces 1:1 daemon-free from typed fixtures
  (cache-seed + a thin fetch override + a seeded SSE stream; **not**
  MSW). One self-contained `intent.html` per surface; tsc
  compile-time drift-guard (a real interface change breaks the twin
  build); ~2-step per-slice authoring loop; production-isolated
  dev-only target. Build any surface with
  `TWIN_ROUTE=<route> npm run twin:build`.
- **`rig scope` intent-visual slot** — `rig scope slice create`
  scaffolds a slice `README.md` that includes an
  `## Intent visual` slot with `[change.diff]` + the
  `TWIN_ROUTE=<route> npm run twin:build` rebuild command. A non-visual
  slice gets an explicit `N/A` line, so the slot is *present-or-N/A*
  per slice.
- **Visual intent → proof convention** — adopted alongside the
  harness: visual intent is approved as a twin mockup before build,
  then proven against a real screenshot at the end. The convention's
  twin-data-freshness condition (every brand-new UI surface has a twin
  fixture; data shapes match real) is folded into the release-close
  gate.

### Topology + For-You

- **Topology edge-flow animation** — when a real queue handoff or
  `rig send` travels an edge (source → destination), the edge animates
  in the handoff direction, brightens while live, then settles static
  after TTL. Honesty-contract held: animates **only** on a real
  queue / send signal (no workflow DAG, no ambient motion);
  un-signaled edges stay static; reduced-motion omits the flow.
  Topology-graph surface only.
- **For-You phone restyle + real-data fidelity** — the Dashboard
  restyle reaches For-You as a phone-friendly cards layout with an
  altitude-dial level filter (All activity → Highlights → Needs you,
  card set changes per level). The internal `source.type` wrapper
  string no longer renders in feed cards, and the vestigial
  storytelling-preview band above the filtered feed is removed (it
  ignored the level filters and visually masked the restyle).

### Reliability + correctness

- **Fresh-install daemon-start fix** — `@hono/node-ws` is now declared
  in the CLI package (the CLI vendors the built daemon and does not
  depend on `@openrig/daemon`, so its `package.json` must mirror the
  daemon's runtime deps for a global `npm install -g @openrig/cli` to
  resolve them). A monorepo-hoisting blind spot meant the dev host
  worked while a fresh install failed; a packaging-completeness gate
  now fails the test suite if a daemon runtime dep is not mirrored in
  the CLI.
- **Table / hybrid terminal-action reliability** — terminal-action
  errors surface as a visible cmux/auth alert instead of silent
  failure on the topology Table view + the hybrid surface; the Table
  terminal now opens cleanly.
- **Queue-item human-readable summary** — additive nullable `summary`
  field on queue items (migration 044). `rig queue create` +
  `rig queue handoff` gain `--summary` with WARN-on-author (warns if
  missing; does **not** hard-break existing callers — phased
  enforcement). The agent-speak body remains the source of truth and
  is inspectable in the queue-item drawer. The Story-row label prefers
  the authored summary and degrades cleanly to the body fallback for
  pre-0.4.1 items.
- **Seat-scoped post-compaction restore** — Claude Code seats get a
  scoped, idempotent restore packet derived from the JSONL transcript;
  the post-compact agent rebuilds its working mental model from
  durable evidence rather than fuzzy recall.
- **Table-view crash fix** — null-safe TanStack filter + an
  `ErrorBoundary` on all three topology Table mounts (host / rig /
  pod) so a single bad row no longer takes down the page; regression
  test added.
- **Dashboard route refresh** — cosmetic visual overhaul of the
  welcome / home launcher route (paper-draft launcher grid + six
  cards + Field Environment + drafting footer). Pure cosmetic; the
  six launcher routes + card test-ids preserved.

### Conventions adopted

- **Visual intent → proof** — the digital-twin convention is adopted
  with three ratified parts: derive-from-source twin (no MSW), one
  `intent.html` per surface, and twin-data-freshness as the residual
  release-close check.
- **Release-durability close — AC-9 (skill-cascade integrity)** —
  `rig skill audit` clean for the shipping set, every drift edge
  green, and bundled plugin reconciled to canonical modulo declared
  transform. Condition 2 (drift edges) remains advisory until the
  shipping cascade checker lands; conditions 1 and 3 are hard from
  this release.
- **Release-durability close — AC-10 (twin-data-freshness)** — the
  twin's component look is 1:1 by construction; the gate covers only
  the two residuals construction does not — dummy dataset
  representativeness (caught mechanically by twin build) and
  net-new-route fixture coverage.

## Behavior + Compatibility

- **Migrations** — additive only. Migration 044 adds a nullable
  `queue_items.summary` column; pre-0.4.1 items degrade to the
  body-fallback label. Pre-0.4.1 daemons that omit 044 also degrade
  cleanly via a `detectQueueColumn`-guarded read path. Existing v0.4.0
  databases upgrade by running `rig daemon start` on the new daemon.
- **`rig queue create` / `rig queue handoff` — `--summary`** —
  optional; missing summary emits a WARN, never a hard error
  (graceful migration). Hard-require is a future hardening.
- **`rig send` default** — now refuses sends when the target pane is
  at a prompt. `--dangerously-interact --reason "..."` is the explicit
  override path.
- **Cosmetic Project tab renames** — Topology → Workflow (test-id
  stable); Queue folds into Story.
- **For-You feed** — the storytelling-preview band on
  `/for-you` is removed (its mission-progress rollup remains on
  Dashboard / Project).

## Known Limitations + Follow-ons

- **Skill-layer cut depth (slice 04 family)** — the canonical
  skill-layer cut still awaits a decision on the load-bearing skill
  set; the cut does not execute in this release. The 04.3 shipping
  cascade-drift checker is not in this cut.
- **Convention drift control (slice 28)** — partially delivered;
  convention ratified + folded into the release-close gate, but the
  shipping product skill + audit-code are still in flight.
- **Twin capture tooling (slice 11.2)** — the twin harness ships; the
  one-command capture wrapper is in flight.
- **Workspace UX architecture (slice 15)** — the design-discovery
  scope is approved; the remaining mockups are in flight (they gate
  future UI slices, not 0.4.1 shipped surfaces).
- **For-You terminal phone-UX (slice 27)** — the For-You restyle and
  real-data fidelity are shipped; the embedded terminal's phone-UX
  (keyboard / touch / width / performance) is a non-gating polish
  check, not yet phone-verified.
- **`rig send` interactive-prompt guard — hook-primary path** — the
  fallback capture-pane guard is fully proven; the hook-primary
  `PermissionRequest` path lands with this release and rides its full
  proof on the next iteration.
- **`rig auth` follow-on (B1 secret-boundary)** — the adversarial
  hardening already merged into `rig auth` closes the symlink +
  hardlink escapes; a deeper structured follow-on is tracked
  separately, not in this cut.
- **Inherited cascade-metadata hygiene** — the 0.4.0 release
  carry-forward (skill-layer `missing_provenance` +
  `missing_verified` audit findings) remains the documented residue;
  bulk fix lives in a separate post-0.4.0 skill-hygiene work item.
